# Excalidraw 界面白屏问题诊断报告

**生成日期**: 2026-04-22  
**项目路径**: `C:\Users\Administrator\.openclaw\workspace\Excalidraw`

---

## 📋 问题描述

程序启动时，界面初始显示正常（能看到功能按钮），随后立即变成一片白色，无法进行任何操作。

---

## 🔍 问题分析

### 1. 项目技术栈

| 组件 | 技术 |
|------|------|
| 前端框架 | React 18 + TypeScript |
| 构建工具 | Vite 5 |
| 桌面端 | Electron |
| 绘图库 | @excalidraw/excalidraw |

### 2. 启动流程分析

```
main.ts → createWindow() 
        → 加载 index.html 
        → 执行 src/main.tsx 
        → 渲染 App 组件 
        → 加载 Excalidraw 组件
```

### 3. 初步诊断

根据代码审查，发现以下几个可能导致白屏的原因：

#### 🔴 原因 1: Vite 开发服务器未正确启动

**文件**: `electron/main.ts` (第 87-92 行)

```typescript
if (VITE_DEV_SERVER_URL) {
  log.info('Loading dev server:', VITE_DEV_SERVER_URL)
  mainWindow.loadURL(VITE_DEV_SERVER_URL)
} else {
  log.info('Loading production build')
  mainWindow.loadFile(join(__dirname, '../dist/index.html'))
}
```

**问题**: 如果 `VITE_DEV_SERVER_URL` 环境变量未正确设置，程序会尝试加载 `dist/index.html`，但该文件可能引用了错误的资源路径。

**检查方法**:
```powershell
# 检查 dist 目录是否存在
Test-Path "C:\Users\Administrator\.openclaw\workspace\Excalidraw\dist"

# 检查 dist/index.html 内容
Get-Content "C:\Users\Administrator\.openclaw\workspace\Excalidraw\dist\index.html"
```

#### 🔴 原因 2: Excalidraw 组件初始化失败

**文件**: `src/App.tsx` (第 178-195 行)

```typescript
<Excalidraw
  excalidrawAPI={(api: any) => setExcalidrawAPI(api)}
  gridModeEnabled={gridEnabled}
  onChange={handleElementsChange}
  UIOptions={{
    canvasActions: { ... },
    tools: { image: false },
  }}
  langCode="zh-CN"
  viewModeEnabled={false}
  zenModeEnabled={false}
/>
```

**问题**: 如果 `@excalidraw/excalidraw` 包加载失败或初始化出错，会导致整个画布区域变成空白。

#### 🔴 原因 3: CSS 样式冲突导致内容不可见

**文件**: `src/index.css` 和 `src/App.css`

```css
/* index.css 第 249-257 行 */
.excalidraw-wrapper {
  width: 100% !important;
  height: 100% !important;
}

.excalidraw {
  width: 100% !important;
  height: 100% !important;
}
```

**问题**: CSS 中的 `width: 100%` 和 `height: 100%` 需要父容器有明确的高度。如果父容器 `.canvas-area` 的高度为 0 或计算错误，Excalidraw 组件会被压缩成不可见。

查看 `App.css` 第 163-170 行：
```css
.canvas-area {
  flex: 1;
  position: relative;
  overflow: hidden;
  background: #fafafa;
  min-width: 0;
  min-height: 0;
}
```

`min-height: 0` 在 flex 布局中可能导致高度计算问题。

#### 🔴 原因 4: CSP (Content Security Policy) 限制

**文件**: `index.html` (第 4-14 行)

```html
<meta http-equiv="Content-Security-Policy" content="default-src 'self' 'unsafe-inline' 'unsafe-eval' http://localhost:* https://localhost:*; script-src 'self' 'unsafe-inline' 'unsafe-eval' http://localhost:* https://localhost:* blob:; style-src 'self' 'unsafe-inline' http://localhost:* https://localhost:*; img-src 'self' data: blob: http: https:; font-src 'self' data: http: https: blob:; connect-src 'self' ws://localhost:* http://localhost:* https://localhost:* https://unpkg.com; worker-src 'self' blob:;" />
```

**问题**: CSP 策略可能阻止了某些必要的资源加载。

## 🧪 排查建议

### 步骤 1: 确认启动模式

运行程序时，检查日志输出：

```powershell
# 查看 Electron 日志
Get-Content "$env:APPDATA\Excalidraw Desktop\logs\main.log" -Tail 50
```

- 如果看到 `Loading dev server: http://localhost:5173` → 开发模式
- 如果看到 `Loading production build` → 生产模式

### 步骤 2: 检查开发服务器状态

如果是开发模式，确认 Vite 服务器是否正在运行：

```powershell
# 检查端口是否被占用
netstat -ano | Select-String "5173"
```

### 步骤 3: 使用 DevTools 诊断

1. 在程序运行时按 `F12` 打开开发者工具
2. 查看 **Console** 标签页，寻找红色错误信息
3. 查看 **Network** 标签页，检查是否有资源加载失败

### 步骤 4: 检查 React 组件状态

在 Console 中执行：
```javascript
// 检查 #root 元素内容
document.getElementById('root').innerHTML.length

// 检查 Excalidraw 组件是否渲染
document.querySelector('.excalidraw')
```

### 步骤 5: 检查窗口尺寸

在 Console 中执行：
```javascript
// 检查 canvas-area 尺寸
const canvasArea = document.querySelector('.canvas-area')
console.log('canvas-area:', canvasArea.getBoundingClientRect())

// 检查 excalidraw-wrapper 尺寸
const wrapper = document.querySelector('.excalidraw-wrapper')
console.log('excalidraw-wrapper:', wrapper ? wrapper.getBoundingClientRect() : 'NOT FOUND')
```

## 🎯 最可能的原因

根据"能看见功能按钮，但随后变白"的症状分析：

**最大可能性：Excalidraw 组件初始化失败**

1. 页面初始渲染正常显示 React 组件（菜单栏、工具栏等）
2. 当 Excalidraw 组件开始初始化时发生错误
3. 错误导致组件渲染失败，canvas 区域变成空白

**建议优先检查**:
1. `@excalidraw/excalidraw` 包是否正确安装
2. 包版本是否与代码兼容
3. 是否有控制台错误信息
