# Excalidraw 图形引擎需求支撑报告

**——面向AI电子黑板项目的技术选型论证**


## 一、为什么需要慎重选择图形引擎？

图形引擎是电子黑板系统的核心基础设施，它决定了“我们能画什么、怎么画、画出来什么样、以及未来能走多远”。一个不合适的图形引擎，后期改弦更张的成本极其高昂。以下将从**商业可行性、技术安全性、功能特点、AI支持程度、行业对标**五个维度，论证选择 Excalidraw 作为图形引擎的战略价值。


## 二、商业可行性：完全放开商用，无需付费

Excalidraw 采用 **MIT 开源许可证**，这是目前全球对商用最友好的授权协议之一：

- **完全免费商用**：可以自由使用、复制、修改、合并、发布、再授权，甚至销售软件副本，无需向原作者支付任何费用，也不必开放自己项目的源代码
- **合规成本低**：唯一的要求是在产品文档或关于页面中保留原始版权声明，注明“本产品部分依赖于 Excalidraw，遵循 MIT 许可证”即可，无需额外审批
- **与传染性协议的区别**：不同于 GPL 等传染性协议——后者要求使用了其代码的衍生作品也必须强制开源，MIT 完全没有这种限制
- **法律风险可控**：MIT 的作者免责条款保护了贡献者，对项目使用方而言，专利风险在前端工具领域极低，通常不构成实质性障碍

这意味着，选择 Excalidraw 可以零授权成本进行商业开发和部署，享有完全的产品自主权。


## 三、技术架构优势分析

### （一）手绘风格的认知友好性

底层的图形渲染使用了手绘风格库（**rough.js** ）。这种手绘风格并非单纯的审美选择，在教育和协作场景中具有实际价值：

- **降低心理门槛**：手绘风让图形看起来“不那么正式”，降低创作者的完美主义压力，鼓励团队成员大胆参与修改和讨论
- **突出内容本质**：让注意力集中在“表达的想法”而非“画得是否精美”上，对教学场景尤为适用
- 每一个图形元素的“潦草程度”都可参数化调节，灵活适配不同场景需求

### （二）开放的数据格式，为AI处理奠定基础

数据存储采用 **基于 JSON 的开放格式**，每个图形元素都被抽象为包含 type、位置、样式等属性的 JSON 对象。

这意味着：
- 任何 AI 系统都可以直接通过生成标准 JSON 来在画布上创建图形，不需要复杂的程序化绘图接口
- 数据天然适合序列化和网络传输，便于多人协作与后端存储
- 支持完整的导入/导出功能，可以随时将图形数据迁移到其他系统

### （三）灵活的导出能力

 支持多种格式导出，包括 **PNG、SVG** 等主流格式，同时支持剪贴板操作和系统文件保存。无论是嵌入教学课件还是存档留痕，都能无缝衔接。

## 五、AI 支持程度分析

这是本报告的核心关注点。Excalidraw 在 AI 集成方面的开放程度远超同类竞品，已形成三种成熟路径：

### （一）AI 驱动的自然语言绘图

用户只需用自然语言描述需求（如“画一个微服务架构图，包含API网关、用户服务、订单服务和数据库”），系统便能通过调用大语言模型（LLM）解析语义，自动生成图形框架并注入画布。

典型代表是 **Smart Excalidraw**，它在开源社区已获得超过 2,000 颗星标，支持：
- 一句话生成 20+ 种图表类型（流程图、架构图、时序图、ER 图、思维导图等）
- AI 根据描述自动选择最合适的图表类型
- 独创的智能连接算法，自动优化箭头布局，避免线条交叉混乱

地址：https://github.com/liujuntao123/smart-excalidraw-next

### （二）AI 代理直接操控画布

通过 **MCP（模型上下文协议）**，AI 助手可以直接操控 Excalidraw 画布，实现“AI 画画、人类看”的全新模式。具体能力包括：

- **完整的元素级操控**：AI 可以创建、修改、删除、查询画布上的每一个图形元素
- **迭代式优化**：AI “画完→审视→调整→再审视”，像人类一样反复迭代精进图表
- **布局工具**：AI 自动对齐、分布、编组/解组图形元素
- **截图反馈**：AI 可以通过截图查看当前画布状态，形成视觉反馈闭环


**URL**:https://www.bilibili.com/video/BV1XvZtBLEDa/?vd_source=60788a4587c619057f174181230a8886

### （三）自主可控的 AI 部署方式

Excalidraw 的 AI 能力并非“绑架”在某个特定大模型上。它通过插件机制调用外部 LLM API，用户可以自由选择底层 AI 服务（ChatGPT、Claude、国产大模型等），随时切换模型或自定义提示词模板，避免被单一AI服务商锁定。

同时支持自托管部署，使用自有服务器的 LLM API Key，完全掌控数据隐私。


## 六、AI电子黑板行业的图形引擎现状

目前市场上 AI 电子黑板产品使用的图形引擎方案主要分为以下几类：

### （一）自研引擎路线（以华为学海白板、科大讯飞为代表）

- **华为学海白板**基于鸿蒙原生 ArkGraphics 2D 构建，拥有笔迹预测算法（书写延迟低于30ms）、智能图形识别、手写公式自动转 LaTeX 等教育专用能力，但技术栈绑定鸿蒙生态
- **科大讯飞AI黑板**融合了3D模型剖分引擎、动态数形转换、虚拟人辅教等技术，侧重于硬件+软件+AI的整体方案交付
- **共性局限**：这些方案多为各自厂商的封闭生态，核心技术细节不公开，跨平台能力有限，第三方难以集成或二次开发

### （二）Web白板开源引擎

在 Web 端白板领域，主要有以下方案：

| 方案 | 核心特点 | 许可证 |
|------|---------|--------|
| **Fabric.js** | 成熟稳定的 Canvas 封装库，生态丰富，被 BoardX 等项目用作底层引擎 | MIT |
| **Plait** | 面向一体式白板的插件化框架，支持自定义渲染和交互，基于 Angular | — |
| **DGM.js** | 支持“智能形状”的无限画布库，React 无头组件，支持多页 | GPLv3 |
| **Excalidraw** | 手绘风格渲染 + 开放 JSON 格式 + 原生 AI 集成 | MIT |

**Tldraw**:https://tldraw.dev/starter-kits/overview

### （三）行业对比总结

当前 AI 电子黑板市场的图形引擎选择呈现两极分化：
- **头部大厂**（华为、讯飞）走自研封闭路线，功能强大但生态锁定严重，不适合跨平台产品
- **开源社区**方案中，Excalidraw 是唯一同时具备成熟Canvas渲染能力、手绘风格、开放数据格式、原生AI支持且MIT商用友好的方案

从行业趋势来看，AI+白板正在成为热点方向。近年来涌现了大量 AI白板工具（Lucidspark、jeda.ai、whiteboard.ai 等），AI 辅助绘图已从“锦上添花”变为“核心能力”。Excalidraw 在这一趋势中占据先机——其 MCP 集成和自然语言绘图能力已在多个项目中得到验证。


建议在立项阶段利完成 Excalidraw + AI 能力的技术原型验证，以最短时间跑通“自然语言描述 → AI 生成图形 → 画布可编辑”的端到端流程，为后续产品规划提供坚实的技术事实依据。

## 技术方案（未来技术栈）

### **智教平台**转`Eletron应用`

####  转技术栈的背景是什么？

`Eletron应用` 支持跨终端，包括`windows`, `macos`, `linux`

而目前我们用户多数使用`windows`，目前国产`麒麟`,`统信`用户逐年增长。

目前电子黑板功能单一，转技术栈成本很小。

这次新增面向AI+电子黑板，以及一些基础学科工具，如果不转技术栈，以后转技术栈需要全部重新开发。成本过于高昂。

#### 打算怎么干

第一阶段只把现有黑板功能转到`Eletron应用`。
第二阶段把主工具条功能也转到`Eletron应用`。
第三阶段把云端功能也转到`Eletron应用`。

#### 人员配合

分2组人，一组负责第一阶段现有黑板功能转到Eletron应用（2个迭代）
另一组负责新增矢量图形功能。
#### 面临的挑战

1：目前Electron 技术栈相对成熟，虽然有AI的加持，但是需要人为兜底。团队需要快速成长，才能满足要求比较高的前端交互方案。

2：在复杂的交互下，AI发挥的能力有限。需要认为的语言描述，目前还是个挑战。

3：目前研究的结果，侧重于矢量图形+Ai功能（手绘公式识别，手绘图形识别，AI绘图，AI集成）相关的研究。如果后续开发其他学科（物理、化学、生物等），需要加入其他技术方案（JS开源库）来进行实现。
